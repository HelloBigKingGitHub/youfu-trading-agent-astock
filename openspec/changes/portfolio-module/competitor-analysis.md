## 本项目 (tradingagents-astock) 差异化建议

**项目定位**：A 股多 Agent 长期投研引擎（7 个 Analyst + Bull/Bear 辩论）。  
**portfolio module 应该是什么**：不是券商 App，不是社交 App，是 **"投研 × 实盘"闭环工具**。

| 维度 | 借鉴对象 | 本项目实现策略 |
|---|---|---|
| 录入方式 | 全部 | 手动为主（用户用例确认）；未来可加"从 Bull/Bear 报告 BUY/HOLD/SELL 一键加自选" |
| 事件类型 | 东方财富 + 雪球 | 买入/卖出/分红/再投/打新 5 类，覆盖个人长线场景 |
| 成本计算 | 东方财富 | 移动加权平均为主 + 用户覆盖成本价按钮（复权处理放到 P2） |
| 盈亏统计 | 有知有行 | XIRR + 年化 + 最大回撤 + 夏普（不学东方财富的"净值相减"伪收益率） |
| 资产配置 | 有知有行 | 行业 + 板块 + 大类（用户手动加"现金/海外/债券"占位） |
| 风险指标 | 有知有行 | Beta / 夏普 / 最大回撤（基础实现，不学 Choice 付费版） |
| 业绩归因 | 有知有行 + 项目本身 | Brinson 模型 + **与 Bull/Bear 报告联动**："我持有 600595 5000 股，成本 5.80，最近 Bull 报告说 BUY，模型给出信号..." |
| 同步 | 全部 | 单一文件 `~/.tradingagents/portfolio/positions.json`，未来加云同步 |

**最大差异化**：跟 Bull/Bear 报告联动——这是其它 4 个 App 都没有的（它们都不做投研）。具体联动点：
1. BUY/SELL 信号 → 自动创建止盈/止损预警
2. 模型信号变化（HOLD → BUY）→ Tab 1 顶部调仓推送横幅
3. LLM 生成报告时调 `get_my_position(ticker)` 看到自己的持仓

## Non-goals 的支撑论证

| 我们不做 | 为什么不做（参考竞品） |
|---|---|
| 券商实盘同步 | 雪球做了一半，用户反馈"支持的券商少"是痛点；本项目"零 cookie"原则 |
| 融资融券 / 期权 / 港美股 | 东方财富做了全套但 UI 复杂度高；MVP 个人用户用不到 |
| 社交化大 V 跟随 / 实盘公开 | 雪球的核心，但本项目定位是"投研引擎"不是"投资社区" |
| OCR 截图识别 | 4 个 App 都没官方支持 |
| 持仓脱敏分享 | P3 候选；MVP 不需要 |
| 价格预警推送 | 已有 `st.session_state` + `st.toast` 临时方案；P3 单独做 scheduler |
